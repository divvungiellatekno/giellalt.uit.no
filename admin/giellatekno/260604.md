# Server

Vi har opplevd nedetid på gtweb pga. overbelastning. 

Problemet var ei spørjing på analyse, frå eit finsk datasenter som sendte ein og same streng mange (30 000) gonger.

Løysing så langt:
    
- ip-avgrensing på cgi-bin (10 i minuttet for same ip, det er kanskje litt lite?)
- avgrensing på cgi-bin (uavhengig av ip-nummer) på 10 i sekundet

Vi følgjer opp dette og ser om det hjelper og om det er for strengt.

Vi bør endre feilmeldinga frå "503 service temporarily unavailable" til "429 Too Many Requests".

Vi kan også vurdere filter for å hindre bot-bruk.
